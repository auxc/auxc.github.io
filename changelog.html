Introducing the first usable version of cobweb. The website has basic search functionality to browse the index, and a frontpage featuring all available tags.

The goal of the project is to create a place where a small group of friends can share and browse interesting bits of the broader internet. Development will focus on improving link sharing, search, organization and archiving features.

All the underlying tech is naive and expected to be replaced with something robust once I'm done experimenting with features.

The name, cobweb, implies a path less traveled. The wooden attic of Web 3.0, filled with items some people thought were cool or useful at one point.

Search

  1. Displays all items from the index that match the search terms.
  2. Spaces between input words are treated as logical AND. If any words do not match at all, an item is not considered a search result.
  3. Page order is based on partial match score calculated using Levenshtein Distance. Close matches in header and description text are valued more than ones in tags and hidden keywords.
  4. Search can be further narrowed down by specifying tags using the following syntax: tag1: tag2: tag3: (...)

Frontpage

  1. All tags present in the index are displayed as links on the frontpage when no search terms are provided.

Link bot

  1. Users access the bot through a private group chat.
  2. Links and metadata are picked up from the conversation and inserted into the index.

April 6th, 2024
Version 0.3

The first experiment with tag hierarchy. The goal is to allow a large amount of specific identifiers associated with each page, making searching and navigation problems easier to solve.

Large number of tags are expected to be a challenge in two ways: to display correctly in the frontend, and to keep accurate in the backend with tag and page combinations growing rapidly in a real deployment.

Frontpage

  1. Tags can now be nested. Nested tags show up when the parent is a part of the search.
  2. Tags added to the search are now displayed inside the input box and can removed by clicking or pressing backspace.
  3. Added a Home button on the left side of the search input box. Currently it allows clearing the search and going back to the home page. In the future, this button will hold the options menu as a drop-down list.

Version 0.5 üß¨ tagweb

The first tag experiment predictably wasn't very usable. Setting up tag relations inside the contents of each index entry proved to be hard to think about and unmaintainable even at a reasonably small index size.

The approaches the hierarchy as an emergent property of the index. The solution follows two simple rules:

  1. Display only the tags that are relevant to the currently ongoing search.
  2. Tag order in any UI where they are expected to be browsed defaults to sorting by the number of results a click would yield.

Maintenance is still a problem to be solved, but the goal is clear when editing an entry: keep the tags and descriptions accurate without worrying about the hierarchy.

Frontpage

  1. Relevant tags are now displayed at all times below the search bar to help browse the index by mouse and narrow down the search.
  2. Tags that would produce 0 results or no change are not shown.
