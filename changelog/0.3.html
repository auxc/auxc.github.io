Version 0.3

The first experiment with tag hierarchy. The goal is to allow a large amount of specific identifiers associated with each page, making searching and navigation problems easier to solve.

Large number of tags are expected to be a challenge in two ways: to display correctly in the frontend, and to keep accurate in the backend with tag and page combinations growing rapidly in a real deployment.

Frontpage

  1. Tags can now be nested. Nested tags show up when the parent is a part of the search.
  2. Tags added to the search are now displayed inside the input box and can removed by clicking or pressing backspace.
  3. Added a Home button on the left side of the search input box. Currently it allows clearing the search and going back to the home page. In the future, this button will hold the options menu as a drop-down list.
